# Анализ `input_softitek.xlsx` для последующей генерации ST-кода

В книге обнаружены листы:
- `Инструкция`
- `PLC`
- `Phase`
- `AI`
- `RTD`
- `DI`
- `DO`

Это соответствует ожидаемой схеме «параметры проекта + таблицы сигналов/каналов».

## 2) Ключевые параметры из листа `PLC`
Из таблицы параметров прочитаны значения:
- `count AI = 11` (число модулей AI)
- `count RTD = 7` (число модулей RTD)
- `count DI = 7` (число модулей DI)
- `count DO = 3` (число модулей DO)

- `count Dt = 23` (номер последней TE/DT по схеме)

- `QFD/QF state = False`

## 4) Что это означает для генератора ST

1. **Можно строить массивы фиксированного размера** на основе `PLC`:
   - `AI[1..11, 1..8]`
   - `RTD[1..7, 1..4]`
   - `DI[1..7, 1..16]`
   - `DO[1..3, 1..16]`
   - `LinePhase[1..32]`

2. **Нумерация каналов не обязательно по порядку в исходной таблице** (это явно указано в `Инструкция`).
   Поэтому генератор должен:
   - читать строку,
   - разбирать `(Модуль, Канал)`,
   - класть тег в точный индекс массива, а не «по порядку строк».

3. **DI/DO могут содержать служебные теги и резерв**:
   - встречаются значения вида `резерв`,
   - спец-теги из инструкции: `auto`, `manuon`, `rkn`.
   Следовательно, в генераторе нужно либо:
   - отдельная ветка обработки спец-тегов, либо
   - единый словарь тегов с последующей нормализацией/маршрутизацией.

4. **Фазировка хранится строками вида `А`, `В`, `А,В`, `С,В,А`**.
   Для ST-кода лучше на этапе генерации сразу преобразовывать это в enum/битовую маску:
   - `A=1`, `B=2`, `AB=3`, `C=4`, `AC=5`, `BC=6`, `ABC=7`.

## 5) Минимальные правила валидации перед генерацией
Рекомендуется валидировать входной Excel перед генерацией ST:
1. Количество уникальных модулей в листах равно `count ...` из `PLC`.
2. Каналы в пределах допустимого диапазона:
   - AI: `1..8`
   - RTD: `1..4`
   - DI/DO: `1..16`
3. Нет дубликатов `(модуль, канал)` в одном листе.
4. Для `Phase` есть ровно `count lines` записей и линии непрерывны.
5. Обрабатывать «пусто/прочерк/резерв» как необязательный канал без падения генератора.

## 6) Черновая модель выходного ST
Практически удобный подход:
- Сгенерировать секцию `VAR_GLOBAL` с константами размеров из `PLC`.
- Сгенерировать структуры привязки (имя тега + модуль + канал).
- Сгенерировать массивы привязок отдельно для `AI/RTD/DI/DO`.
- Сгенерировать таблицу фаз `LinePhase[LineNo]`.

Пример (идея, не финальный синтаксис под конкретную среду):
```st
VAR_GLOBAL CONSTANT
    CntAI : INT := 11;
    CntRTD : INT := 7;
    CntDI : INT := 7;
    CntDO : INT := 3;
    CntLines : INT := 32;
END_VAR
```

## 7) Вывод
Текущий Excel выглядит **консистентным** и пригодным для автоматической генерации ST. 
Основной акцент для генератора: **не полагаться на порядок строк**, а использовать точную адресацию `(модуль, канал)` и явную валидацию перед выводом кода.
