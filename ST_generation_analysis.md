# Повторный анализ исходных данных и программ генерации ST

Дата проверки: **2026-03-27**.

## 1) Что входит в исходные данные
Входной файл: `input_softitek.xlsx`.

Используемые листы:
- `PLC` — проектные константы;
- `AI` — токовые сигналы;
- `RTD` — температурные точки;
- `DI` — дискретные входы;
- `DO` — дискретные выходы;
- `Phase` — фазировка линий.

## 2) Фактические параметры из `PLC`
Повторная проверка через текущий парсер показала:
- `count_ai = 10`
- `count_rtd = 7`
- `count_di = 7`
- `count_do = 3`
- `count_lines = 32`
- `count_dt = 23`
- `first_group = 17`

Производная величина:
- `count_modules = 27` (10 + 7 + 7 + 3)

## 3) Согласованность таблиц сигналов
- `AI`: 80 точек, 10 модулей, дубликатов `(модуль, канал)` нет.
- `RTD`: 28 точек, 7 модулей, дубликатов нет.
- `DI`: 112 точек, 7 модулей, дубликатов нет.
- `DO`: 48 точек, 3 модуля, дубликатов нет.
- `Phase`: 32 записи, линии покрыты полностью `1..32`.

Вывод: данные выглядят консистентными для текущей логики генерации.

## 4) Анализ программ

### 4.1 Парсинг Excel (`st_codegen/xlsx_plc_reader.py`)
- Чтение реализовано без `pandas/openpyxl`: `.xlsx` открывается как ZIP и разбирается XML.
- Поддерживается `sharedStrings`, поиск листа по имени, чтение значений по колонкам.
- Формат входа строгий: пустые/нечисловые значения silently пропускаются для точек.

Сильные стороны:
- минимум зависимостей;
- предсказуемый и быстрый путь чтения табличных данных.

Риски:
- нет строгой валидации диапазонов каналов (например, AI должен быть `1..8`);
- для AI/DI/DO/RTD пропущенные строки не дают явной ошибки;
- ошибки структуры листов обнаруживаются только косвенно.

### 4.2 Генерация GVL (`st_codegen/gvl_generator.py`)
- Шаблон корректно использует константы из `PlcConfig`.
- На текущих данных должен формироваться `count_ai=10`, `gc_count_modules=27`.

### 4.3 Генерация Proc_IO (`st_codegen/proc_io_generator.py`)
- Формируются:
  - контроль RTD + таймеры ошибок;
  - токи линий из AI;
  - состояния QFD/KM из DI;
  - команды на DO;
  - фазировка;
  - массив ошибок модулей.
- Логика KM c суффиксами `.2`, `.3` и т.д. корректно переиспользует токи/состояния базовой линии.

Риски:
- при отсутствии ожидаемых тегов (`km`, `qfd`, `qfds`, `auto`, `manu`) код генерируется частично без явной ошибки;
- `count_dt=23` означает использование только первых 24 RTD-точек из 28 (это сейчас допустимо, но не проверяется как контракт).

## 5) Сверка артефактов в репозитории
Повторная генерация показала отсутствие отличий:
- `GVL.txt` совпадает с результатом `generate_gvl.py`;
- `Proc_IO.txt` совпадает с результатом `generate_proc_io.py`.
